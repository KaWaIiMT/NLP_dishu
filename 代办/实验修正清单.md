# 待办：实验修正

> 创建日期：2026-05-30
> 来源：迭代-001

---

## 背景

Task B 的分析脚本误将"白话版标注数据"（中文自然语言文本）当作"地书数据"进行了分析，导致 TTR、Zipf、熵等结果不可靠。

**修正方向**：将分析对象从白话版中文文本切换为原始《地书》图形标注数据中的标注值。

---

## 实验修正清单

### EXP-FIX-01：基于 literal_gloss 的齐夫定律分析

| 项目 | 内容 |
|------|------|
| **输入数据** | `素材/《地书》标注数据/《地书》标注任务1-词标注/*.txt` + `句标注/*.txt` |
| **提取字段** | `annotations` 中 `task_id: "literal_gloss"` 的 `value` |
| **分析内容** | 词频排名分布、齐夫斜率、R² |
| **对照组** | 21个自然语言语料库（来自 `任务组A-数据准备/Task3-语料库采样/采样结果/`） |
| **输出** | 地书 vs 各自然语言的齐夫对比表 + 双对数图 |
| **依赖** | 无 |
| **优先级** | 🔴 高 |

### EXP-FIX-02：基于 literal_gloss 的信息熵与词汇丰富度分析

| 项目 | 内容 |
|------|------|
| **输入数据** | 同上，`literal_gloss` 值 |
| **分析内容** | 信息熵、归一化熵、TTR、RTTR、MTLD |
| **对照组** | 21个自然语言语料库 |
| **输出** | 地书 vs 自然语言的熵和词汇丰富度对比表 |
| **依赖** | 无（可与 EXP-FIX-01 合并） |
| **优先级** | 🔴 高 |

### EXP-FIX-03：POS-like Category 分布对比分析

| 项目 | 内容 |
|------|------|
| **输入数据** | `素材/《地书》标注数据/《地书》标注任务1-词标注/*.txt` + `句标注/*.txt` |
| **提取字段** | `annotations` 中 `task_id: "pos_like_category"` 的 `value`（数组） |
| **分析内容** | 名词性/动词性/形容词性等类别的频率分布 |
| **对照组** | 现代中文语料库的 POS 分布（可使用 jieba 分词 + 词性标注） |
| **输出** | 地书 POS类别 vs 自然语言词性 的分布对比图 |
| **依赖** | EXP-FIX-01, EXP-FIX-02 |
| **优先级** | 🟡 中 |

### EXP-FIX-04：篇章关系类型分布分析

| 项目 | 内容 |
|------|------|
| **输入数据** | `素材/《地书》标注数据/《地书》标注任务2-句标注/*.txt` |
| **提取字段** | `annotations` 中 `task_id: "discourse_relation"` 的 `value`（单选） |
| **分析内容** | 时间先后/因果/转折/并列等篇章关系的频率分布 |
| **对照组** | 可考虑与 PDTB（Penn Discourse TreeBank）中文版对比 |
| **输出** | 地书篇章关系分布 |
| **依赖** | EXP-FIX-01 |
| **优先级** | 🟡 中 |

### EXP-FIX-05：形态特征分布分析

| 项目 | 内容 |
|------|------|
| **输入数据** | `素材/《地书》标注数据/《地书》标注任务1-词标注/*.txt` + `句标注/*.txt` |
| **提取字段** | `annotations` 中 `task_id: "morphological_features"` 的 `value`（数组） |
| **分析内容** | 时态/体貌/否定/复数/强调等形态特征的频率分布 |
| **对照组** | 自然语言的形态特征分布（中文少→可与英文等对比） |
| **输出** | 地书形态特征分布 vs 自然语言对比 |
| **依赖** | EXP-FIX-01 |
| **优先级** | 🟢 低 |

### EXP-FIX-06：重新运行零样本风格分类

| 项目 | 内容 |

| **注意事项** | 当前 Gemini API 为模拟模式，需切换为真实 API |
| **依赖** | EXP-FIX-01 |
| **优先级** | 🟡 中 |

在 `代办/` 目录下创建具体的实验脚本后，使用 `/experiment-agent` 的 `run` 模式执行：

```
/experiment-agent run <实验脚本路径>
```
