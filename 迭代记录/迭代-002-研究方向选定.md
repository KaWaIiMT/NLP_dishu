# 迭代-002：研究方向选定（2026-05-30）

### 背景
迭代-001 完成了数据溯源，明确了可用的数据资产（原始标注数据46,000+条）。基于作业要求（"研究问题独特性及意义"）和数据独特优势，进行了多方向探讨。

### 讨论过的方向
- **A**：设计"语言性"量化评估框架
- **C**：标注分歧作为歧义性的量化窗口
- **E**：设计语言 vs 演化语言的统计指纹
- **F**：图形作为跨语言翻译中介
- **G**：歧义热力图 — 方向C的可视化深化
- **H**：视觉特征→语义功能的映射规律（计算符号学）

### 选定方向：A + C+G

**A（语言性评估框架）** — 论文的方法论骨架：
- 提出一套多维度评估指标体系（Zipf拟合度、信息熵、POS分布多样性、篇章结构复杂度、语义稳定性等）
- 对《地书》和多种自然语言进行评分
- 框架本身可复用（Emoji、交通标志、人造语言等）

**C+G（标注分歧 + 歧义热力图）** — 论文的核心创新实证：
- 利用多标注者数据，量化每个语言学维度、每种语义类型的标注者间一致性
- 构建"歧义热力图"：行=语言学维度，列=语义类型，颜色=分歧程度
- 回答"图形语言的歧义集中在哪些维度、哪些语义类型"

### 两者的互补关系
- A 回答 **"有多像语言"**（宏观打分）
- C+G 回答 **"哪里不像、为什么"**（微观定位）
- A 中"语义稳定性"（inter-annotator agreement）作为框架的一个维度，C+G 是该维度的深度展开

### 对论文的影响
- 论文从"描述性统计报告"升级为"方法论框架 + 实证发现"
- 创新点清晰：首次提出图形语言评估框架 + 首次系统量化图形语言歧义分布
- 符合作业要求：经典NLP方法（A的统计指标）+ LLM方法（可作为A框架中的"盲测验证者"）

### 本轮范围
- 不包含 LLM 分析（按用户决定，先聚焦纯统计分析）
- 投稿目标：中文期刊

### 补充决策：方向F的处理（2026-05-30）

方向F（图形作为跨语言翻译中介）被认为是好想法，但经过数据核实：
- `crosslinguistic_equivalents` 实际标注量为 **0**（标注体系中定义了该任务但无人被分配/无人填写）
- 因此F无法作为实证维度纳入本文

**处理方式**：
- 在论文结论章中作为"框架可扩展性"的一部分提及（D7维度）
- 作为未来工作方向，后续补充数据后可做独立论文
- 不增加正文篇幅，仅作为前瞻性展望
