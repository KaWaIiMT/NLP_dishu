## 迭代-001：文献调研与数据溯源（2026-05-30）

### 背景
项目已有初步实验产出（Task A 数据准备、Task B LLM分析），但研究者本人对这些工作的具体含义尚未完全理解。本次迭代的目标是：
1. 搞清楚"数据是什么"
2. 搞清楚"分析方法做了什么"
3. 搞清楚"已有结果是否可靠"

### 关键发现：项目中存在两类完全不同的标注数据

| | 原始《地书》图形标注 | 白话版标注 |
|---|---|---|
| **位置** | `素材/《地书》标注数据/` | `地书实验结果/清洗后数据/` |
| **标注对象** | 《地书》的图形符号本身 | 学生用中文写的《地书》白话翻译故事 |
| **data_type** | （无此字段） | `"natural_language_text"` |
| **文件数** | 词级366 + 句级342 = **708个文件** | 词级84 + 句级100 = **184个合规文件** |
| **标注总量** | **46,494条**（词级27,652 + 句级18,842） | 未统计 |
| **标注维度** | 全部10个语言学维度 | 每个学生只负责2个任务 |
| **图形本体** | ❌ 不在JSON中（在标注工具内） | ❌ 不适用 |
| **数据状态** | 原始导出，可直接使用 | 经过清洗（剔除了104个不合规文件） |

### 问题一：Task B 的初步结果不可靠

**问题描述**：
- Task B 的分析脚本（`style_feature_extractor.py`）把"白话版标注数据"当作了"地书数据"来分析
- 白话版数据的 `raw_text` 是中文自然语言（学生写的故事），不是地书图形语言
- 因此得出的 TTR=0.659, Zipf=-0.426, 熵=10.815 等结果，分析的是"地书的翻译文本"而非"地书本身"

**判定**：这些结果应标记为不可靠/需重做。

**原因**：
- 类比：如果你想研究"手语的语言学特征"但分析的是"手语的文字翻译稿"，那得出的结论无法回答关于手语的问题

### 问题二：原始标注数据缺少图形视觉形态

**问题描述**：
- 原始《地书》标注数据的 JSON 中，`target_id` 指向 `grp_xxxxxx_xx` 等图形组ID
- 但图形的视觉形态（形状、颜色、复杂度等）不在导出数据中——标注者是在标注工具里看图形做标注的

**影响**：
- ✅ 可以分析：语义内容（literal gloss）、语法角色（POS）、形态特征、篇章关系、语义基元
- ❌ 无法分析：视觉象似性、视觉复杂度、符号组合的空间关系

**决策**：当前阶段聚焦于"基于标注值的语言学分析"，后续如有需要再补充视觉维度数据。

### 做出的决策

1. **分析方法论修正**：后续所有对"地书"的分析，Token 来源必须是原始标注数据中的标注值（如 `literal_gloss`），不能用白话版中文文本
2. **论文诚实声明**：论文中需要明确说清楚"分析对象是标注值而非图形本体"，这不是缺陷，而是研究方法的透明性
3. **重新实验计划**：需要基于原始标注数据重跑宏观语言学分析
4. **数据架构明确**：
   - **原始《地书》标注** → 用于"地书图形语言 vs. 自然语言"的核心对比
   - **白话版标注** → 可用于"白话翻译文本 vs. 其他自然语言文体"的补充对比
   - **自然语言语料库**（21种）→ 对照组

### 对论文方向的影响

初步将论文论点方向调整为：
> 基于计算语言学方法，对《地书》图形语言的标注体系进行多维度量化分析，检验其在词频分布、信息熵、语法类别分布等维度上与自然语言的异同。

核心贡献仍然是：**首次对《地书》进行系统性的计算语言学定量分析**。
