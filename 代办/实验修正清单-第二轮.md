## Bug 1：自然语言语料的分词方式错误 🔴

### 问题描述

`exp_fix_01_zipf.py` 第 109 行：

```python
words = list(text)   # ← 把字符串按字符拆开！
```

这导致：

- **英文/德文/法文**：每个字母被当作一个"词"。top "词" 是 "e", "t", "a", " "（空格）。数据完全不可用。
- **中文**：每个汉字被当作一个"词"（如"的"、"一"、"是"）。虽然比英文好一些，但也不是正确的分词——"图书馆"应该是一个词，不是"图"+"书"+"馆"。
- **日文**：同理，混合了汉字、假名、数字，tokenization也是错的。

### 修复方案

**中文**（新闻、小说、古文、文学）：使用 jieba 分词

```python
import jieba
words = list(jieba.cut(text))
```

**英文/德文/法文**：使用 nltk 或简单的正则分词

```python
import re
words = re.findall(r"[a-zA-Zà-üÀ-Üß]+", text.lower())
# 这会匹配真正的单词，不是单个字母
```

**日文**：使用 nagisa 或 fugashi，或者简单用 MeCab。如果太复杂，本轮可以先用正则 `re.findall(r'[぀-ゟ゠-ヿ一-鿿]+', text)` 提取日文词素。

### 影响范围

- EXP-FIX-01：Zipf 对比表（英文/德文/法文数据需重跑）
- EXP-FIX-02：熵和 TTR 对比表（同上）

**POS、篇章关系、形态特征（EXP-FIX-03/04/05）不受影响**，因为它们分析的是《地书》的标注值，不涉及自然语言分词。

## Bug 2：TTR 对比的样本量不一致 🟡

### 问题描述

- 《地书》literal_gloss：**10,538** tokens
- 自然语言语料：**50,000** tokens（脚本按字符数截断，实际是约50,000个字符）

TTR（Type-Token Ratio）对文本长度极度敏感——文本越长，分母越大，TTR 越低。这不是语言的固有属性差异，而是样本量差异。

**举例**：如果把《地书》数据扩充到50,000 tokens，它的TTR一定会从0.43大幅下降。公平的对比必须在相同的token数量下进行。

### 修复方案

**方法一：降采样（推荐）**

把所有语料统一降采样到《地书》的规模（10,538 tokens），然后计算 TTR。多次随机采样取平均值以消除偶然性。

```python
import random
def downsampled_ttr(word_list, target_size, n_trials=100):
    tt_ratios = []
    for _ in range(n_trials):
        sample = random.sample(word_list, target_size)
        tt_ratios.append(len(set(sample)) / target_size)
    return np.mean(tt_ratios), np.std(tt_ratios)
```

**方法二：使用长度鲁棒指标**

RTTR（Root TTR = Types / sqrt(Tokens)）已经在第一轮数据中计算了，它部分校正了长度影响。论文中可以同时报告降采样TTR和RTTR。

**建议**：两个方法都用。降采样TTR作为主指标，RTTR作为验证。

### 影响范围

- EXP-FIX-02：TTR 对比表需要重跑
